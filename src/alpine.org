default, without having to configure it in every project. Alpine is my
go-to base image because it's small and fast to pull. The timezone

* Setting the timezone on Alpine
  :PROPERTIES:
  :ID:       e4e79d12-b0d3-4121-981c-90de5dfd4246
  :END:

  On Alpine, timezone data comes from the =tzdata= package. We install it,
  copy the zoneinfo file, then remove the package to keep the image small.

  [[https://docs.dagger.io/api/reference#definition-Container][.with_exec]] runs a shell command inside the container — the
  equivalent of =RUN= in a Dockerfile. Here it installs =tzdata=, copies
  the zoneinfo file, and removes the package in a single step so the
  final image stays small.

  #+begin_src python
  @function
  def alpine_set_tz(self, ctr: dagger.Container) -> dagger.Container:
      """Set timezone on an Alpine container (uses Lib.timezone)."""
      tz = self.timezone
      return ctr.with_exec([
          "sh", "-c",
          "apk --quiet add --update tzdata"
          f" && cp /usr/share/zoneinfo/{tz} /etc/localtime"
          f' && echo "{tz}" > /etc/timezone'
          " && apk --quiet del tzdata",
      ])
  #+end_src

  We can verify the timezone is correctly set:

  #+NAME: alpine_timezone
  #+begin_src bash
  dagger call alpine with-exec --args="cat","/etc/timezone" stdout
  #+end_src

* A base Alpine container with optional extra packages
  :PROPERTIES:
  :ID:       fcb8570e-2515-4a0f-a703-4b6ed673464a
  :END:

  Sometimes I need additional tools. The =alpine= target gives me a
  timezone-ready Alpine with an easy way to add packages.

  [[https://docs.dagger.io/api/reference#definition-Container][dag.container()]] creates a new empty container — the starting point
  for every pipeline step. [[https://docs.dagger.io/api/reference#definition-Container][.from_]] sets the base image, like =FROM= in
  a Dockerfile. We chain =with_exec= to install any extra packages the
  caller requested.

  #+begin_src python
  @function
  def alpine(self, extra_packages: list[str] = ()) -> dagger.Container:
      """Alpine with timezone set and optional extra packages."""
      ctr = dag.container().from_(f"alpine:{self.alpine_version}")
      ctr = self.alpine_set_tz(ctr)
      if extra_packages:
          ctr = ctr.with_exec(["apk", "--quiet", "add"] + list(extra_packages))
      return ctr
  #+end_src

* A non-root user for safer workflows
  :PROPERTIES:
  :ID:       53e17e2e-2890-4b31-a125-fd02a0265c3a
  :END:

  Running as root in containers is risky. I want a default user ready to
  go, so I don't have to repeat user-creation boilerplate.

  #+begin_src python
  @function
  def alpine_user(
      self,
      extra_packages: list[str] = (),
      groups: list[str] = (),
  ) -> dagger.Container:
      """Alpine with a default user."""
      ctr = self.alpine(extra_packages=extra_packages)
      return self.use_user(ctr, groups=groups)
  #+end_src

* Timezone artifacts for other containers
  :PROPERTIES:
  :ID:       4bd886bd-2213-484a-900e-c3cac400e5a6
  :END:

  Some containers (like distroless) can't run =apk=. I need to extract the
  timezone files from Alpine so I can copy them in.

  [[https://docs.dagger.io/api/reference#definition-Container][.file(path)]] pulls a single file out of a container as a [[https://docs.dagger.io/api/reference#definition-File][File]]
  artifact. [[https://docs.dagger.io/api/reference#definition-Directory][dag.directory()]] creates a new empty Directory and
  [[https://docs.dagger.io/api/reference#definition-Directory][.with_file]] adds files into it. Together they let us bundle timezone
  files extracted from Alpine so other containers can import them.

  #+begin_src python
  @function
  def alpine_tz(self) -> dagger.Directory:
      """Extract /etc/localtime and /etc/timezone from Alpine as artifacts."""
      ctr = self.alpine()
      return (
          dag.directory()
          .with_file("localtime", ctr.file("/etc/localtime"))
          .with_file("timezone", ctr.file("/etc/timezone"))
      )
  #+end_src

* Python with a user and virtualenv
  :PROPERTIES:
  :ID:       9ad0ca24-648d-4092-8acf-2143f4a27040
  :END:

  For application development, I want a full setup: Alpine + Python + a
  non-root user + a virtualenv ready at =/app/venv=.

  [[https://docs.dagger.io/api/reference#definition-Container][.with_workdir]] changes the container's working directory — like
  =WORKDIR= in a Dockerfile. Setting it to =/app= means subsequent
  commands and the virtualenv land in the right place.

  #+begin_src python
  @function
  def alpine_python_user_venv(
      self,
      extra_packages: list[str] = (),
      groups: list[str] = (),
      packages: list[str] = (),
      work_dir: str = "/app",
  ) -> dagger.Container:
      """Alpine with python, user, and a virtualenv."""
      ctr = self.alpine_python(extra_packages=extra_packages)
      ctr = self.use_user(ctr, groups=groups)
      ctr = ctr.with_workdir(work_dir)
      return self.python_venv(ctr, base=work_dir, packages=packages)
  #+end_src
