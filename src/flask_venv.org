I often build small Flask APIs. Instead of repeating the venv setup in
every project, I want a pre-built virtualenv with Flask installed that
I can copy into my final image.

* A Flask virtualenv as a directory artifact

  This returns the =/app/venv= directory with Flask (and optional extra
  packages) already installed. I can then copy this into any container.

  [[https://docs.dagger.io/api/reference#definition-Container][.directory(path)]] extracts a directory from the container as a
  [[https://docs.dagger.io/api/reference#definition-Directory][Directory]] artifact. Here it pulls out the fully-built virtualenv so
  it can be copied into a different, leaner final image without
  repeating the install steps.

  #+begin_src python
  @function
  def flask_venv(self, packages: list[str] = ()) -> dagger.Directory:
      """Debian python user venv with flask, exported as directory artifact."""
      ctr = self.debian_python_user_venv(packages=["flask"] + list(packages))
      return ctr.directory("/app/venv")
  #+end_src
