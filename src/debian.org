Some of my services need libraries or tools that are only well-packaged
on Debian. I want Debian slim containers with the same defaults as my
Alpine ones: French timezone, no unnecessary packages pulled in, and
easy user setup.

* Disabling automatic recommends

  Debian's apt installs recommended packages by default, which bloats
  images. I disable this globally so every =apt install= stays lean.

  #+begin_src python
  @function
  def debian_no_auto_install(self, ctr: dagger.Container) -> dagger.Container:
      """Disable apt recommends and suggests."""
      return ctr.with_exec([
          "sh", "-c",
          'echo \'APT::Install-Recommends "0";\' > /etc/apt/apt.conf.d/01norecommend'
          " && "
          'echo \'APT::Install-Suggests "0";\' >> /etc/apt/apt.conf.d/01norecommend',
      ])
  #+end_src

* Setting the timezone on Debian

  Debian ships timezone data by default, so it's simpler than Alpine: just
  symlink the right zoneinfo file.

  #+begin_src python
  @function
  def debian_tz_fr(self, ctr: dagger.Container) -> dagger.Container:
      """Set timezone to Europe/Paris on a Debian container."""
      return (
          ctr
          .with_exec(["rm", "/etc/localtime"])
          .with_exec(["ln", "-sf", "/usr/share/zoneinfo/Europe/Paris", "/etc/localtime"])
      )
  #+end_src

* Cleaning up apt caches

  After installing packages, I clean up to keep the image small.

  #+begin_src python
  @function
  def debian_apt_cleanup(self, ctr: dagger.Container) -> dagger.Container:
      """Clean apt caches."""
      return ctr.with_exec([
          "sh", "-c",
          "apt-get --quiet clean && rm -rf /var/lib/apt/lists/*",
      ])
  #+end_src

* A base Debian container

  This is the Debian equivalent of my Alpine target: slim image with
  timezone set, no recommends, and optional extra packages.

  #+begin_src python
  @function
  def debian(self, extra_packages: str = "") -> dagger.Container:
      """Debian slim with Europe/Paris timezone, no auto-install, and optional extra packages."""
      tag = f"{self.debian_version}.{self.debian_min_version}-slim"
      ctr = dag.container().from_(f"debian:{tag}")
      ctr = self.debian_no_auto_install(ctr)
      ctr = self.debian_tz_fr(ctr)
      if extra_packages:
          ctr = ctr.with_exec([
              "sh", "-c",
              "{ apt-get --quiet update"
              f" && apt-get --quiet install --yes {extra_packages}"
              " ; } > /tmp/log 2>&1 || { cat /tmp/log; exit 1; }",
          ])
          ctr = self.debian_apt_cleanup(ctr)
      return ctr
  #+end_src

* Python with a user and virtualenv on Debian

  For Python services that need Debian-specific libraries (e.g., for
  native extensions), I want the same venv setup as Alpine but on Debian.

  #+begin_src python
  @function
  def debian_python_user_venv(
      self,
      extra_packages: str = "",
      groups: str = "",
      packages: str = "",
      work_dir: str = "/app",
  ) -> dagger.Container:
      """Debian with python, user, and a virtualenv."""
      ctr = self.debian(extra_packages=f"python3-venv {extra_packages}".strip())
      ctr = self.use_user(ctr, groups=groups)
      ctr = ctr.with_workdir(work_dir)
      return self.python_venv(ctr, base=work_dir, packages=packages)
  #+end_src

* Extracting the Europe/Paris timezone file

  Distroless containers can't install packages. I extract the timezone
  file from Debian so I can copy it into distroless images.

  [[https://docs.dagger.io/api/reference#definition-Container][.file(path)]] extracts a single file from the container as a [[https://docs.dagger.io/api/reference#definition-File][File]]
  artifact. This lets us grab the zoneinfo file and later inject it into
  images that have no package manager.

  #+begin_src python
  @function
  def debian_europe_paris(self) -> dagger.File:
      """Extract the Europe/Paris localtime file from Debian."""
      tag = f"{self.debian_version}.{self.debian_min_version}-slim"
      return (
          dag.container()
          .from_(f"debian:{tag}")
          .file("/usr/share/zoneinfo/Europe/Paris")
      )
  #+end_src

* Creating a Python venv

  A reusable helper to create a virtualenv with =--system-site-packages=
  and optionally install packages into it.

  [[https://docs.dagger.io/api/reference#definition-Container][.with_env_variable]] with =expand=True= lets the value reference
  existing variables — here we prepend the venv's =bin/= directory to
  =$PATH= so that =python= and =pip= resolve to the venv copies without
  hard-coding the rest of the path.

  #+begin_src python
  @function
  def python_venv(
      self,
      ctr: dagger.Container,
      base: str,
      packages: str = "",
  ) -> dagger.Container:
      """Create a Python venv with --system-site-packages and optionally install packages."""
      ctr = ctr.with_exec([
          "python3", "-m", "venv", "--system-site-packages", f"{base}/venv",
      ]).with_env_variable("PATH", f"{base}/venv/bin:$PATH", expand=True)
      if packages:
          ctr = ctr.with_exec([
              "sh", "-c",
              f"{base}/venv/bin/python -m pip --quiet install --upgrade {packages}",
          ])
      return ctr
  #+end_src
